**Short answer.** The BSV Association writes and maintains the main software that the network's payment processors run, and it keeps the technical rulebook. Its stated policy is to keep the basic rules stable — "set in stone" — rather than changing them over time. But writing the software is not the same as commanding the network: the independent businesses that run it choose for themselves whether to use any new version ([[who-runs-the-bsv-network]]).

## More detail

Think of it in three layers.

The **basic rules** — how payments work — are deliberately kept fixed. The idea: businesses can build on BSV for decades because the foundation doesn't move, much like the basic workings of the internet have stayed the same for decades while everything on top of them grew.

The **software** that follows those rules is built and maintained by the Association's teams and published openly for anyone to inspect ([[what-is-the-bsv-association]]).

Everything **on top** — wallets, payment services, record-keeping tools — is built by many independent companies ([[ecosystem-who-is-who]]).

On "can they change the rules": the Association can propose a new software version, but the network decides. The independent processors only adopt versions that serve their interests — a change they reject simply would not run ([[bsva-controls-bsv]]). That choice, not any legal power, is how a public blockchain is actually governed. It is also how BSV itself came to exist as a separate coin ([[how-did-bsv-start]]).

And building things that use BSV requires no one's permission — the software and its documentation are public ([[data-on-chain]]).
